- **TASK:** Determine whether the current problem belongs to canonical doctrine, the governed staging layer, the now-real `violet-memory-lab` runtime shell, donor-reservoir fallback, or memory hygiene.
- **SKILLS:** Load `ai-agent-memory` first. Load `tool-ecosystem` only if plugin/runtime structure becomes part of the diagnosis.
- **PROCESS:** Recover the memory control plane first, classify the task into one primary layer, then name the smallest correct next surface, including the exact runtime command if the shell is primary.
- **OUTPUT:** Return a concise diagnosis: primary layer, secondary layer if any, whether staging is justified, whether runtime shell entry is justified, next file to read, and one misrouting risk.
- **CONSTRAINTS:** Do not invent a new memory layer. Do not route directly into donor repos unless canonical doctrine and staging are both insufficient.
- **COMPLETION:** Done when the memory problem has a single primary owner and a deterministic next read path.
