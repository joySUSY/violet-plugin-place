# Agentic System Basis Index

## Purpose

Canonical navigation entrypoint for `agentic-system-basis` doctrine inside `developer-tool`.

Use this category when the problem is about:

- harness capability detection
- bootstrap and onboarding flows
- activation and runtime exposure of agentic doctrine
- multi-agent or ensemble governance
- coordinator behavior
- factory or gate-based autonomy models
- AI-team structure, role shaping, or working agreements

This index is not only a document list.
It exists to route readers into the correct agentic-system doctrine lane based on the kind of orchestration or activation pressure they need to resolve.

## Agentic-System Decision Questions

Before choosing a subpage, ask:

1. Is the real pressure about **system model**, **activation**, **bootstrap/capability detection**, **team governance**, or **autonomy/gate design**?
2. Is this still a doctrine-first orchestration question, or already a runtime-shell/tool-ecosystem question disguised as one?
3. Do we need the broad system model first, or are we already certain which specialized lane dominates?
4. Are we trying to classify, activate, coordinate, or enforce?

The doctrine is:
- agentic-system docs are organized by orchestration pressure and activation pressure
- not by whichever keyword around “agents” appeared first
