# AI Agent Memory Index

## Purpose

Canonical entrypoint for the detailed memory and continuity doctrine under `developer-tool/references/ai-agent-memory/`.

Use this category when the task is already known to belong to the memory lane and you need to choose the correct detailed doctrine for:

- recall-before-action
- recall runtime patterns
- search-before-act enforcement
- continuity and interruption recovery
- retrieval lane selection

This index is not only a document list.
It exists to route readers into the correct detailed memory doctrine based on the kind of memory or continuity pressure they need to make explicit.

## Memory Decision Questions

Before choosing a subpage, ask:

1. Is the real pressure about **memory architecture**, **recall discipline**, **retrieval lane selection**, **continuity recovery**, **trigger ownership**, or **advanced/reflex memory**?
2. Is this still a doctrine-first memory question, or already a runtime-shell or governance question disguised as one?
3. Do we need the broad memory model first, or are we already certain which specialized lane dominates?
4. Are we trying to know something before acting, preserve something across interruption, or decide where a memory behavior should live?

The doctrine is:
- memory docs are organized by recall pressure, continuity pressure, and trigger/governance pressure
- not by whichever memory-flavored term appears first
