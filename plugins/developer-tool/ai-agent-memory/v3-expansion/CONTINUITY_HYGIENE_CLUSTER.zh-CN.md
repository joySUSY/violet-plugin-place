# Continuity and Hygiene Cluster

**Authors: Joysusy & Violet Klaudia**

> 连续性负责让工作不断线，卫生治理负责让记忆不腐化。
> Continuity keeps work alive; hygiene keeps memory legible.

## 目的

这份 staged cluster 文档解释 `v3-expansion/` 中 **continuity / hygiene** 这一组的角色。

当 canonical continuity 或 memory-hygiene doctrine 已经完成问题定向，但仍然需要更深的 operational support modules 时，才应该进入这一组。

## 本集群包含的模块

- `goodvibes-memory/`
- `memory-hygiene/`
- `memory-model/`
- `meta-cognition-parallel/`

## 什么时候打开这一组

当真正的问题是：

- memory cleanup / pruning
- memory asset 的 schema / usage support
- 更底层的 memory-model specialist concepts
- 与长会话或 memory-system 行为相关的 meta-cognitive / parallel support patterns

这时才进入这一组。

## 这一组不是什么

这一组不是：

- continuity 的主 doctrine
- recall 的主 doctrine
- interruption recovery 设计的第一入口

这些问题应先回到：

- `../../references/ai-agent-memory/continuity-control-plane.md`
- `../../references/ai-agent-memory/working-state-and-self-reminder-discipline.md`
- `../../references/ai-agent-memory/memory-systems.md`

## 未来更可能的收束方向

与 retrieval cluster 相比，这一组更可能产出未来的 canonical support docs，尤其围绕：

- hygiene discipline
- schema / maintenance behavior
- continuity support 与 memory-model theory 的进一步区分

但其中某些模块因为抽象层次较高，可能会长期保持 niche staged 状态。
