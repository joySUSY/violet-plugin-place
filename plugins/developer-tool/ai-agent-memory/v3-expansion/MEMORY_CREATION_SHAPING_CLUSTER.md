# Memory Creation and Shaping Cluster

> Shape memory deliberately before asking the runtime to remember it.
> 先设计记忆的形状，再要求运行时去记住它。

## Purpose

This staged cluster doc explains the **memory creation / shaping** side of `v3-expansion/`.

Use this cluster only when the canonical memory references have already oriented the task and the remaining need is about:
- creating memory-related skills or artifacts
- shaping memory structures or retrieval-learning workflows
- exploring meta-memory techniques that are not yet cleanly flattened into canonical doctrine

## Modules in This Cluster

- `advanced-memory-skill-creator/`
- `agent-memory-skills/`
- `memory-systems/`
- `memory-retrieval-learning/`

## When to Open This Cluster

Open this cluster when the question is no longer:
- “what should I recall?”
- “which lane of history should I use?”
- “how does continuity work?”

and instead becomes:
- “how should a memory-oriented skill or artifact be created?”
- “what shape should a memory system or retrieval-learning asset take?”
- “what adapted materials exist beyond the canonical refs for memory creation or shaping?”

## What This Cluster Is NOT

This cluster is not:
- the first-line doctrine for recall
- the continuity control plane
- the default source of truth for memory architecture

For those, return to:
- `../../references/ai-agent-memory/recall-before-act.md`
- `../../references/ai-agent-memory/memory-systems.md`
