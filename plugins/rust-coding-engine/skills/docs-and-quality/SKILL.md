# Rust Docs and Quality

## Steps

1. Read `../../references/quality/INDEX.md`.
2. Read `../../references/quality/rust-quality-testing-benchmarking-documentation.md`.
3. If style and lint posture is central, read `../../references/quality/rust-style-and-lint-governance.md` and `../../references/quality/rust-workspace-lint-pipeline-discipline.md`.
4. If public API or rustdoc quality is central, read `../../references/quality/rustdoc-mastery.md`, `../../references/quality/rust-doc-examples-discipline.md`, and `../../references/quality/rust-public-api-documentation-and-trait-surface-discipline.md`.
5. Use `../../rules/style/INDEX.md` and `../../rules/testing/INDEX.md` for exact subdomain routing.
6. Keep docs/examples/lints aligned with runtime reality.

## Core Rule

Quality is not decoration. In Rust, docs, examples, lints, and tests are part of the executable contract.
