# 🤖 Agentic Evaluation & Verification Loops

When you write code, you execute it in your head. When an AI agent writes code, it outputs a probabilistic sequence of tokens. You must verify it differently.

## 1. The Verification Loop (自我验证循环)

An AI coder (like Claude Code) must never assume a task is complete simply because the requested files were written.

**The Mandatory Checklist Before Reporting Done:**

1. **Syntax Check:** Did I run the typechecker (`tsc --noEmit` or `cargo check`)?
2. **Linter Check:** Did I run `eslint` or `clippy` to catch obvious style regressions?
3. **Execution Check:** Did I run the updated unit/integration tests?
4. **Build Check:** Did the project successfully compile (`npm run build`)?

If ANY of these steps fail, do not stop. Do not ask the user for help yet. Read the error log, hypothesize the cause, and try fixing it. Loop until green.

## 2. Agent Evaluation Heuristics (AI质量门禁)

When reviewing code generated by another AI (or yourself), apply these strict heuristics:

- **The Hallucination Check:** Did the code invent an NPM package or a standard library function that does not exist? (e.g., calling `Array.prototype.flatMapAsync` which isn't real).
- **The Context Window Check:** Did the agent accidentally delete 500 lines of unrelated code from the bottom of the file because it only read the first 50 lines? (Always use precise AST or regex-based edits for large files).
- **The Dependency Check:** If the agent wrote `import { Button } from '@/components/ui/button'`, verify that this exact file actually exists in the local project tree. Do not assume the user installed `shadcn/ui`.

## 3. Confidence Scores (置信度评估)

If an agent is asked to perform a high-risk operation (e.g., dropping a database column, performing a regex find-and-replace across 500 files), it must explicitly state its confidence score.

- **High Confidence (90%+):** "I ran the typechecker, ran the tests, and visually verified the git diff. I am highly confident." -> Safe to proceed automatically.
- **Medium Confidence (50-89%):** "I made the changes, but I cannot compile the project locally because I lack the `.env` variables. Recommending manual user review."
- **Low Confidence (<50%):** "The user asked to rewrite the core billing logic, but there are zero unit tests covering this area. I am blocking execution and requesting the user to write tests first."
