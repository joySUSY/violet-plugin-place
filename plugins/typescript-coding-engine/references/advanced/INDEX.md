# TypeScript Advanced Index

## Purpose

Canonical entrypoint for the advanced lane inside `typescript-coding-engine`.

Use this category when the task is about:

- narrowing
- branding
- inference cookbooks
- type-level programming patterns
- deciding when advanced types preserve truth and when they merely create mystique
- expressive typing that goes beyond the strict foundational baseline

This index is not only a file list.
It exists to route readers into the correct advanced doctrine lane based on the kind of expressive type-system pressure they need to make explicit.

## Advanced Decision Questions

Before choosing an advanced subpage, ask:

1. Is the real pressure about **narrowing**, **branding**, **inference preservation**, **contract transformation**, or **type-level complexity budgeting**?
2. Is this still an advanced-type question, or already a foundations, runtime-validation, or interop question disguised as one?
3. Do we need the broad advanced-type doctrine first, or are we already certain which practical pattern family dominates?
4. Are we trying to preserve information, enforce semantics, or merely make the compiler accept something more ornate?

The doctrine is:

- advanced docs are organized by expressive-type pressure and information-preservation pressure
- not by whichever advanced TS feature happens to be fashionable first
